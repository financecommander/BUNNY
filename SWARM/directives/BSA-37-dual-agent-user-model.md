# BSA-37 — Dual-Agent User Model (Jack + Jenny)

**System:** Calculus AI Platform
**Authority:** BUNNY Core + BUNNY Shield
**Component:** Dual-Agent Personal Intelligence Layer
**Directive Type:** Enterprise Agent + Personal Agent Coordination

---

## 1. Concept

Each Calculus user may have two agents:

| Agent | Role | Domain |
|---|---|---|
| Jack | Professional assistant | Company / enterprise |
| Jenny | Personal assistant | Life / household / personal |

This creates a dual-agent ecosystem per person.

```
User
│
├── Jack (Enterprise AI)
│        │
│        ├ company systems
│        ├ project intelligence
│        ├ document systems
│        ├ workflows
│
└── Jenny (Personal AI)
         │
         ├ personal tasks
         ├ family coordination
         ├ home automation
         ├ health / finance
```

Both agents run on the same swarm infrastructure, but under separate permission domains.

## 2. Separation of Data Domains

The most important rule is strict domain separation.

### Jack Access
- Company documents
- Project systems
- Enterprise data
- Internal APIs
- Organizational intelligence

### Jenny Access
- Personal calendars
- Home automation
- Personal finances
- Health tracking
- Family schedules

Jenny cannot access enterprise systems by default.

## 3. Controlled Cross-Agent Collaboration

Jack and Jenny may cooperate only in limited ways.

### Scheduling
- **Jenny:** Family calendar
- **Jack:** Work calendar
- **Together:** Conflict resolution suggestions

**Example:** Jenny: family dinner 6 PM. Jack: board meeting 6 PM. Result: conflict resolution suggestion.

### Travel Coordination
- **Jack:** Business travel, meeting schedules
- **Jenny:** Personal travel logistics, packing reminders, home prep

### Health vs Work Balance
- **Jenny monitors:** Sleep, wellness, family time
- **Jack monitors:** Workload, meetings, deadlines
- **Together:** Schedule adjustment, workload reduction, focus time suggestions

## 4. Agent Personality Model

### Jack Personality
- Professional, structured, strategic, task-oriented
- Behaves like: chief of staff, operations partner, research assistant

### Jenny Personality
- Friendly, supportive, casual, personal
- Behaves like: life organizer, household assistant, wellness companion

## 5. Shared Memory Bridge

The system may maintain a controlled memory bridge.

**Bridge Rules:**
- Only metadata shared
- No sensitive documents
- No private financial or health data

**Example Shared Data:**
- Calendar conflicts
- Travel plans
- Major deadlines

Everything else remains isolated.

## 6. Device Deployment

The dual-agent system works well with Calculus AI nodes.

| Device | Primary | Secondary |
|---|---|---|
| Work device | Jack | Jenny (secondary) |
| Personal phone | Jenny | Jack (notifications only) |
| Home server node | Jenny | Home automation + personal memory |

## 7. Example Daily Interaction

### Morning Briefings

**Jenny:**
> Good morning. You slept 7 hours. Family dinner tonight. Gym reminder.

**Jack:**
> Good morning. Three meetings today. Two priority emails. Financial report deadline.

## 8. Example User Commands

### Work
- "Jack, prepare the Ryegate briefing."
- "Jack, pull the Excel model from Drive."

### Personal
- "Jenny, remind me to pick up groceries."
- "Jenny, schedule a dentist appointment."

### Hybrid
- "Jack and Jenny, check for schedule conflicts this week."

## 9. Privacy Model

**Security Layers Must Enforce:**
- Enterprise isolation
- Personal data protection
- Agent permission boundaries

**BUNNY Shield Must Ensure:**
- Jack cannot access personal health/finance data
- Jenny cannot access company systems

## 10. Why This Model Is Powerful

Most AI systems only provide one assistant. This architecture provides:

**Enterprise intelligence (Jack) + Life intelligence (Jenny)**

Together this becomes a complete personal operating system.

For Calculus employees this means:
- One AI for work
- One AI for life
- Both coordinated

## 11. Strategic Value for Calculus

This architecture allows Calculus to build something very unique:
- Enterprise AI OS
- Personal AI OS
- Distributed AI node network
- Agent marketplace

Very few systems attempt this.

## 12. Directive Activation

Directive BSA-37 is now active.

All dual-agent coordination systems must comply with the separation, privacy, and collaboration rules defined in this directive.
