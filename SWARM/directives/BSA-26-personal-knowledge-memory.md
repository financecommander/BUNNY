# BSA-26 — Personal Knowledge Memory Directive

**System:** Calculus AI Platform
**Authority:** BUNNY Core (Personal Intelligence) + BUNNY Shield (Privacy & Security)
**Component:** Personal Knowledge Memory Layer
**Directive Type:** Long-Term Personal Memory, Idea Capture, and Knowledge Recall

---

## 1. Purpose

This directive establishes the Personal Knowledge Memory system within the Calculus AI platform.

Jack must maintain a persistent memory of information relevant to individual users:
- Store personal notes and ideas
- Remember past conversations
- Track research topics
- Organize personal knowledge
- Recall information when needed

The objective is to transform Jack into a long-term intellectual partner.

## 2. Personal Knowledge Framework

```
Knowledge Capture
    ↓
Classification
    ↓
Storage
    ↓
Indexing
    ↓
Retrieval
    ↓
Review & Update
```

Each entry generates a Personal Knowledge Record.

## 3. Knowledge Capture

**Capture Methods:**
- Voice dictation
- Chat notes
- Document uploads
- Meeting notes (BSA-16)
- Research summaries

**Example:** *"Jack, remember this idea for the energy project."*

## 4. Knowledge Classification

| Category | Examples |
|---|---|
| Ideas | Concepts, brainstorms, innovations |
| Research topics | Ongoing investigations |
| Project notes | Project-specific context |
| Technical knowledge | Technical solutions, patterns |
| Strategic insights | Market observations, strategic thoughts |

## 5. Knowledge Storage

### Vector Memory

**Purpose:** Semantic search, context retrieval, conversation recall

**Systems:** Qdrant, Weaviate, pgvector

### Personal Knowledge Graph

**Purpose:** Relationships between ideas, project connections, topic networks

**Systems:** Neo4j, ArangoDB

### Document Archive

**Purpose:** Long-form notes, documents, reports

## 6. Knowledge Indexing

| Index Type | Purpose |
|---|---|
| Semantic | Meaning-based search |
| Topic | Category navigation |
| Project | Project-scoped retrieval |
| Timestamp | Chronological access |

## 7. Knowledge Retrieval

**Example Queries:**
- *"Jack, what ideas did we discuss about biomass power?"*
- *"Jack, find my notes from last week's strategy meeting."*

**Retrieval Methods:** Semantic search, keyword search, project-based search

## 8. Contextual Recall

Jack recalls relevant knowledge during conversations:

*User: "Let's revisit the energy project idea."*
*Jack: retrieves past notes and summaries automatically.*

This creates continuity across conversations.

## 9. Idea Development

Jack assists in developing stored ideas:
- Expanding concepts
- Connecting related ideas
- Suggesting research directions

**Example:** *"This idea connects to the research we did on renewable power financing."*

## 10. Knowledge Review

**Review Triggers:**
- Scheduled knowledge reviews
- Project updates
- User requests

**Example:** *"You saved an idea about AI energy modeling six months ago. Would you like to revisit it?"*

## 11. Personal Knowledge Dashboard

Dashboard shows:
- Recent notes
- Saved ideas
- Active research topics
- Knowledge connections

## 12. Integration with Organizational Knowledge

Personal knowledge may be promoted to organizational knowledge:

```
Personal idea → Validation → Organizational knowledge base (BSA-15)
```

## 13. Privacy & Security

- User-level access control
- Encrypted storage
- Private memory isolation
- Visible only to associated user unless explicitly shared

BUNNY Shield enforces these policies.

## 14. Learning from Personal Knowledge

Jack analyzes stored knowledge for patterns:
- Frequent research themes
- Recurring project interests
- Innovation ideas
